Samples file, installation and configuration (Last revision 06/12/2021)

First, before uploading all the content to the server, you must edit some variables that are inside the config.json file.

1) jActionPath:
In this variable you must specify the folder where the jActionLib library is located.

Ex .: "jActionPath": "https://www.yourdomain.com/lib/jaction"

2) jActionFwPath:
In this variable you must specify the folder where the Framework is located.

Note: This variable is only needed if you have downloaded jActionLib and jActionFramework separately, if you have downloaded them merged into one file then this variable should be left blank and only need to specify (jActionPath).

Ex .: "jActionFwPath": "https://www.yourdomain.com/lib/jaction-framework"

3) rootPath:
This variable should generally be blank, it should be used to precede any type of resource loaded in the project. When being blank, the resources will be relative to the initial file of the project index.html. Therefore, if at another time we want to point to another route, we will only have to modify the value to change the route of all the resources at the same time.

Ex .: "rootPath": ""

4) absRootPath:
This variable must contain the absolute path of the folder where the initial file of the project is located, for example samples.

Ex: "absRootPath": "https://www.yourdomain.com/samples"

5) domain:
This variable must contain the address of your domain.

Ex .: "domain": "https://www.yourdomain.com"

6) Meta:
In this variable you can indicate some metas. Basically the metas have been included here to simplify the initial project file index.html, but you can leave the meta variable blank and specify the metas directly on the index.html project page.

Note that if you use the meta icon from config.json the browser will show an error indicating that it cannot find the favicon but the favicon is loaded correctly, the problem is that the browser checks the favicon right at the time of starting index.html and although the favicon is loaded later, the browser marks the error.

Once config.json is configured, upload all the content to the main root of your server in a folder called samples. (Important, the server must have a certificate to work under the https protocol, otherwise the library will not work)
